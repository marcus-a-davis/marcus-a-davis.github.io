Ruby Classes and JavaScript Constructors

JavaScript and Ruby take different approaches to modeling objects that define objects. That is, the syntax for creating objects with no functionality of their own is quite different in the two languages. Let's begin by seeing how you create such objects in JavaScript:

            1  function Philosopher() {
            2    this.jobDescription = "Thinking really hard.";
            3  }
            4      
            5    
            6  my_philosopher = new Philosopher();

This is a constructor function in JavaScript. On line 1 I begin the definition by using the keyword "function" and giving the function a name, Philosopher. On line two I create an intial value for the philosopher object using the "this.property" syntax. All this means is for the object in which the "this" appears, the method "job" is equal to whatever you put on the right side of the equals operator.

On line 6 I create a new instance of the Philosopher function by using the "new" operator. By doing so I've set the jobDescription property I defined on line 2 as "Thinking really hard." In fact, by defining the jobDescription property in the constructor function above, I've defined the jobDescription property for all new instances of the Philosopher class. Theoretically you could set this property for each new instance of the Philosopher object you create whenever you initially define the object. However, by making a constructor function you can save yourself a lot of time and typing.

Let's model this same behavior in Ruby:

            1  class Philosopher
            2    def initialize
            3      @jobDescription = "Thinking really hard."
            4    end 
            5  end
            6
            7  my_philosopher = Philosopher.new

Line 1 of this code creates a new class, or type of object, Philosopher. Line 2 creates an "initialize" method, which is a way to determine what is done each time a new instance of this class is created. Line 3 is where the parallel to JavaSript constructors is most clear, as it defines the variable "@jobDescription" as "Thinking really hard.".

Again, as in JavaScript, the final line of code creates a new instance of the Philosopher class. However, in Ruby the "new" is not a keyword (like "break" or "continue") but a method of the Class class (the class from which all other classes inherit).

Naturally, though the syntax isn't all that different, the behavior of classes in Ruby and constructor functions in JavaScript is not identical. Much of this is due to the difference in focus on object-oriented design between the languages. However, with minor tweaks you can get JavaScript constructor functions to behave very similarly to classes in Ruby.

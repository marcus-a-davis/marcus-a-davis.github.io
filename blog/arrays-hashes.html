Arrays and Hashes

When programming you may want to store a list of information. You could do so by assigning each item in the list to a separate variable. However, this is really inefficient. What you'd need to make it easier is an array, which is a way to store lists in a programming language.

How to create an array: array = [item1, item2, item3]

In general you want to use arrays anytime you want to store lists.If you are storing simple lists like lists of names or a list of your favorite movies arrays are great way to store the data.

However, all lists are not made the same. Some lists are themselves lists of lists. Suppose, for example, you were running a website and you wanted to store usernames and passwords. You would want each username to be unique but your users could use the same passwords. You could store that information in two separate arrays (or, more implausibly, in a single array) and retrieve the information at the same time whenever you needed it. However, that's pretty inefficient and if one entry in either list is ever removed without it's counterpart being removed your entire system would break down.

How to create a hash: hash = {key1: "value1", key2: 2, key3: "value3"}

What you'd want in that situation is a way to store both usernames and passwords in the same place. For that you'd need a hash. This way when trying to determine if a username and a password match you would only have to check one entry as information in hashes is stored as pairs known as keys and values.

logins = {username1: "password1", username2: "password2", username3: "password3"}

Of course, it would be bad practice to keep your passwords stored in a literal form like that but the basic idea is right. If you want to store a list which really consist of two lists with connected entries you should use a hash.
